# UI Refresh Guidelines

## Core Reference

本轮界面优化的首要视觉与结构参考文件是：

- [ui-refresh-preview.html](E:\project\003desk\game\demo\ui-refresh-preview.html)

后续无论是继续迭代 HTML 效果稿，还是落地到 Godot，都应优先参考这个文件里的布局关系、组件轻重、信息层级和视觉风格，而不是回退到当前游戏原型的旧三栏结构。

尤其要重点参考以下方面：

- 战场居中、信息贴边的主布局
- 单层 `deploy` 区而不是多层叠板
- 底部轻量化操作带
- 多车道敌人推进与横向部署的对应关系
- 我方攻击效果的静态视觉表达
- 统一的西方魔幻卡通头像风格

## Overall Direction

目标不是照抄参考手游 UI，而是把当前桌面版原型改造成更成熟、更友好、更像正式产品的横屏战斗界面。

总方向：

- 战场是第一视觉焦点
- 高频信息集中，低频信息边缘化
- 操作闭环清晰，避免用户来回扫视
- 风格统一，避免调试面板感
- 英雄、敌人、图腾、按钮图标都使用同一套美术语言

## Layout Principles

### 1. Battlefield First

- 主视觉中心始终是战场，不再让左右大信息栏抢焦点
- 顶部只保留关卡、波次、少量状态徽记
- 左侧保留少量快捷入口，例如倍速、统计、日志/设置
- 右侧只保留图腾信息和轻量预警

### 2. One-Layer Deploy Area

- `deploy` 区只保留一层，不再叠加独立候选面板
- 随机英雄信息并入 `deploy` 头部，用轻提示而不是大卡片表达
- 部署槽位是主角，提示信息只是辅助

### 3. Bottom Action Strip

- 底部操作区应保持轻、扁、靠底
- 左侧显示金币
- 中间显示核心稳定度，并合并核心等级
- 右侧只保留两个主操作：
  - 图腾强化
  - 召唤英雄

不再保留：

- 独立“核心中枢”卡片
- “刷新候选”操作

## Gameplay-to-UI Mapping

### 1. Random Hero Logic

- 英雄出现逻辑应表达为“随机英雄”
- 不要再强调候选刷新
- UI 上只需提示当前可能出现的随机英雄类型或示例，不要把它做成第二个主面板

### 2. Horizontal Deploy vs Enemy Flow

当前设计原则是：

- 下方是横向部署带
- 战场中应体现多车道推进
- 敌人不能再集中从正中一条竖线推进

正确表达方式：

- 将战场分成多条纵深车道
- 敌人在不同列/不同车道推进
- 我方部署与这些车道形成明确对应关系

## Combat Readability

### 1. Hero Attack Effects

我方攻击效果不能只靠角色站位和飘数字，需要有更美观的视觉提示。

当前建议保留并继续强化的效果类型：

- `beam`
  - 用于远程射线、冰霜法术、能量打击
- `ring`
  - 用于冻结、控制、范围锁定
- `burst`
  - 用于命中爆点、魔法爆裂
- `spark`
  - 用于暴击、火花、雷击触发

这些效果的目的不是完全拟真，而是让玩家第一眼看懂：

- 谁在攻击
- 攻击打向哪里
- 是什么属性或类型的攻击

### 2. Damage Feedback

- 飘字要服务于战斗反馈，而不是满屏乱飘
- 伤害数值应围绕命中点分散
- 配合效果层使用，而不是单独存在

## Visual Style

### 1. Art Direction

角色风格统一为：

- 卡通
- 西方魔幻
- 更写实一些的头像感
- 不是 emoji 占位风格
- 也不是纯抽象图形块

### 2. Avatar System

需要统一风格的对象包括：

- 战场上的英雄头像
- 战场上的怪物头像
- `deploy` 槽位内的小头像
- 图腾头像
- 底部操作图标

原则：

- 所有头像都属于同一视觉体系
- 色相可以区分类别，但造型语言要一致
- 不再混用 emoji、图标字体、临时字符

### 3. Panel Style

继续保持以下样式语言：

- 半透明深色面板
- 柔和高光和描边
- 圆角偏大
- 组件更亲和而不是硬核科幻控制台
- 主按钮颜色明确，功能区分明显

## Current Approved Decisions

以下思路已在当前效果稿中得到确认，应视为后续实现默认方向：

- 单层 `deploy` 结构
- 右侧 `High Threat` 改为纯图标预警
- 底部操作带更轻、更扁、更靠底
- 核心等级并入核心稳定度
- 去掉刷新候选
- 英雄随机出现
- 多车道敌人推进
- 增加我方攻击效果层
- 头像风格向卡通西方魔幻统一

## Implementation Advice

如果后续落地到 Godot，建议优先按下面顺序实现：

1. 先对齐主布局和区域关系
2. 再处理 `deploy` 单层化和底部操作带
3. 再修正战场车道与敌人分布
4. 最后补攻击效果和头像美术风格

不要先做零散美化，再回头改结构，否则会重复返工。

## Priority Reminder

后续任何继续优化，都应先检查：

- 是否仍然符合 [ui-refresh-preview.html](E:\project\003desk\game\demo\ui-refresh-preview.html) 的结构
- 是否仍然保持“战场优先、边缘降噪、底部闭环”
- 是否引入了新的重复面板或多层叠板
- 是否破坏了横向部署与多车道推进的关系

如果和这个参考文件冲突，默认以这个参考文件的样式和结构为准。
